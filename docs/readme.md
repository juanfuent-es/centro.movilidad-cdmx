# Documentación Movilidad CDMX

Esta carpeta contiene toda la documentación estratégica, técnica y conceptual del proyecto.  

*Nada aquí es código productivo: es soporte, contexto y definición*

---

## ¿Qué debe incluir esta carpeta?

### Concepto y Estrategia
- Brief del proyecto
- Objetivos
- Público objetivo
- Alcances y limitaciones
- Problema a resolver
- Propuesta de valor

---

### Dirección Visual
- Moodboards
- Referencias visuales
- Sistema visual
- Lineamientos de arte
- Paleta de color (con valores HEX / RGB / HSL)
- Tipografías y usos

---

### Arquitectura y Estructura
- Mapa del sitio (sitemap)
- Wireframes
- Estructura de navegación
- Jerarquías de información
- Flujos de usuario

---

### Bitácora
- Cambios importantes
- Decisiones técnicas
- Problemas encontrados y soluciones
- Iteraciones del proyecto

---

## Reglas

- No subir archivos pesados innecesarios.
- Mantener los documentos organizados por nombre claro.
- Usar formato `.md` cuando sea posible.
- No mezclar código productivo dentro de esta carpeta.

---

## Objetivo

La carpeta `/docs` debe permitir que cualquier persona:
- Entienda el proyecto rápidamente.
- Pueda continuar el desarrollo.
- Comprenda las decisiones de diseño.
- Tenga contexto sin depender del autor original.
