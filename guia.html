Tipografías

El proyecto usa variables tipográficas y carga de Google Fonts. Puedes sustituir o extender con Adobe Fonts o Font Squirrel.

  • Google Fonts
  • Adobe Fonts (Typekit)
  • Font Squirrel

Variables locales en css/style.css:

:root {
--font-base: 'Roboto', Helvetica, Arial, Lucida, sans-serif;
--font-heading: 'Barlow', Helvetica, Arial, Lucida, sans-serif;
}

body { font-family: var(--font-base); }
h1, h2, h3 { font-family: var(--font-heading); }

Carga por CDN en _includes/head.html:

<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Barlow:...&family=Roboto:...&display=swap" rel="stylesheet">

Ejemplo de uso en HTML:

<h1 class="stroke-white">Título con <span class="color-orange">Barlow</span></h1>

Colores y temas

Se usan variables CSS y utilidades para color y contorno; hay soporte a modo oscuro automático.

Variables en css/style.css:

:root {
--color-asfalto: #212121;
--color-white: #FFFFFF;
--color-stop-rojo: #D32F2F;
--color-metro-naranja: #F47C20;
--color-taxi-rosa: #EC407A;
--color-senal-verde: #2E7D32;
--color-bg: var(--color-white);
--color-text: var(--color-asfalto);
--color-primary: var(--color-stop-rojo);
}

Utilidades disponibles:

.color-orange { color: var(--color-metro-naranja); }
.stroke-white { -webkit-text-stroke: 1px var(--color-white); }
.button.button-pink:hover { background-color: var(--color-taxi-rosa); }

Ejemplo de uso en HTML:

<a class="button button-orange" href="#">CTA Naranja</a>

Skeleton (Grid y baseline)

Se importa Skeleton para grid y tipografía base. Las utilidades propias viven en css/style.css.

Importación en _includes/head.html:

<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/skeleton/2.0.4/skeleton.min.css">

Layout responsivo con grid:

<section class="row">
<article class="six columns card">Columna 1</article>
<article class="six columns card">Columna 2</article>
</section>

Componentes locales (ejemplos):

.hero { min-height: 70vh; display: flex; align-items: center; }
.card { border: 1px solid var(--color-text); padding: 1em; }
.button { font-weight: 800; text-transform: uppercase; }

Scroll y accesibilidad

Scroll personalizado con variables y smooth scrolling respetando preferencias de movimiento.

Definición en css/scroll-bar.css:

:root { --scroll-size: 10px; --scroll-background: var(--color-bg); }
::-webkit-scrollbar { width: var(--scroll-size); height: var(--scroll-size); }
::-webkit-scrollbar-thumb { background-color: var(--scroll-foreground); }
@media (prefers-reduced-motion: no-preference) {
html { scroll-behavior: smooth; }
}

Consejo: Evita forzar parallax/animaciones si el usuario prefiere reducir movimiento.

Estructura y tecnologías

Arquitectura estática con includes y utilidades CSS.

  • Skeleton: grid responsivo con row y columns.
  • Utilidades CSS: variables, colores y helpers color-*, stroke-*, button-*.
  • Font Awesome: iconografía por CDN.

Ejemplo de bloque hero reutilizable:

<header class="hero container">
<h1 class="stroke-white">Movilidad <span class="color-pink">CDMX</span></h1>
<p>Texto descriptivo</p>
<a class="button" href="{{ '/guia.html' | relative_url }}">Comenzar</a>
</header>

Navegación (ver _includes/header.html):

<a class="nav-link" id="mapa-ruta-link" href="{{ '/guia.html' | relative_url }}">
<i class="fa-solid fa-map-location-dot"></i>
<span class="nav-text">Mapa de Ruta</span>
</a>

Font Awesome

Iconografía vía CDN ya incluida en _includes/head.html. Usa las clases de la librería para incrustar íconos.

Importación (ya presente):

<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.4.0/css/all.min.css">

Ejemplos de uso en HTML:

<i class="fa-solid fa-font" aria-hidden="true"></i>
<i class="fa-solid fa-tag" aria-hidden="true"></i>
<button class="button"><i class="fa-solid fa-location-dot" aria-hidden="true"></i> CTA</button>

Accesibilidad: cuando el ícono es decorativo, añade aria-hidden="true". Si transmite información, acompáñalo con texto visible.

Buenas prácticas rápidas

  • Define colores y tipografías solo en variables; usa utilidades para consumo.
  • Usa el grid de Skeleton para layout; componentes propios para estética.
  • Mantén metadatos por página en el front matter; evita duplicar en HTML.
  • Respeta prefers-color-scheme y prefers-reduced-motion.
  • Optimiza accesibilidad: contraste, tamaños legibles y texto alternativo.
